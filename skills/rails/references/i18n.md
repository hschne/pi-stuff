# I18n Patterns

### Translation Files

We keep both Rails translation files and frontend translation files. **Important**: Translations for the frontend (JSON) are automatically generated from the Rails localization files (YAML). **NEVER** modify the JSON localization files directly!

```bash
# Modify
config/locales/de.yml
config/locales/en.yml
# NEVER modify
app/javascript/locales/de.json
app/javascript/locales/en.json
```

For example:

```yaml
de:
  activerecord:
    models:
      project: 'Projekt'
      phase: 'Phase'
```

```json
{
  "activerecord": {
    "models": {
      "project": "Projekt",
      "phase": "Phase",
    }

```

### Rails i18n Structure

Rails i18n follows specific conventions for organizing translations. **Always follow this structure:**

```yaml
en:
  activerecord: # Model-related translations
    models: # Model names (singular)
      user: 'User'
      project: 'Project'
    attributes: # Model attributes (for form labels)
      user:
        name: 'Name'
        email: 'Email'
        password: 'Password'
      project:
        title: 'Title'

  helpers: # Form helpers
    submit: # Submit button text
      user:
        create: 'Sign up'
        update: 'Update Profile'
      project:
        create: 'Create Project'
    label: # Additional label hints
      user:
        password_hint: '12 characters minimum'

  manage: # View-specific namespaced translations
    projects:
      index:
        title: 'Projects'
        new: 'New Project'
      create:
        success: 'Project was successfully created.'

  components: # View-specific component translations
    ideaCard:
      title: 'Idea Card'
```

### When to Use Each Namespace

**1. `activerecord.models`** - Model names (singular)

```yaml
activerecord:
  models:
    user: 'User'
    project: 'Project'
```

Used automatically by Rails in error messages and form labels.

**2. `activerecord.attributes.{model}`** - Form field labels

```yaml
activerecord:
  attributes:
    user:
      email: 'Email'
      password: 'Password'
      name: 'Name'
```

**Why?** Rails form helpers automatically pick these up:

```erb
<%= form_with model: @user do |f| %>
  <%= f.label :email %>
  <%= f.text_field :email %>
<% end %>
```

**3. `helpers.submit.{model}`** - Submit button text

```yaml
helpers:
  submit:
    user:
      create: 'Sign up' # For new records
      update: 'Update Profile' # For existing records
```

Rails automatically uses the correct key based on whether the record is new or persisted:

```erb
<%= form_with model: @user do |f| %>
  <%= f.submit %>  <!-- "Sign up" for new, "Update Profile" for existing -->
<% end %>
```

**4. View-specific namespaces** - Flash messages, titles, static text

```yaml
sessions:
  new:
    title: 'Sign in'
    or: 'OR'
    forgot_password: 'Forgot your password?'
  create:
    success: 'Signed in successfully'
    error: 'That email or password is incorrect'
```

Use for:

- Page titles
- Flash messages
- Links and buttons that aren't form submits
- Static UI text

### Use Lazy Lookups

Utilize Rails "Lazy Lookup" for translation keys whenever possible.
